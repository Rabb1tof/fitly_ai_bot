# Contributing Guide

Правила и рекомендации для разработчиков HealthBot.

## Общие принципы
- Пишите код и документацию на английском/русском — важно сохранять консистентность.
- Соблюдайте существующий стиль кода (анализируйте примеры в репозитории).
- Все PR должны сопровождаться описанием изменений и ссылкой на задачу.
- Старайтесь, чтобы каждый PR был небольшим и решал одну проблему.

## Рабочий процесс
1. Создайте ветку от `main`: `git checkout -b feature/<name>`.
2. Внесите изменения, сопровождая их тестами, если возможно.
3. Запустите `dotnet build` и `dotnet test` перед отправкой PR.
4. Обновите документацию (README, docs/*), если меняется функциональность или API.
5. Откройте PR и дождитесь ревью.

## Стиль кодирования
- Используйте `async/await`, `CancellationToken` во всех асинхронных методах.
- Добавляйте `ILogger<T>` для логирования важных событий и ошибок.
- Избегайте "магических" строк — выносите их в константы/настройки.
- При работе с Telegram разделяйте message и callback handlers.

## Тесты
- Обязательно покрывайте новую логику тестами (unit/integration).
- Не оставляйте `TODO` без задачи — заводите issue.
- Для тестирующих helper-ов смотрите на `HandlerHarness`.

## Документация
- Если меняется архитектура, workflows или требования — обновите соответствующие документы в `docs/`.
- README должен оставаться актуальным для новых разработчиков.

## Безопасность
- Никогда не коммитьте реальные токены/пароли.
- Проверяйте `.gitignore` и `.env`.

## Code review
- Проверяйте декомпозицию изменений, тесты и документацию.
- Давайте конструктивные комментарии и предложения по улучшению.

## Контакты/поддержка
- Основной канал коммуникации — issues и PR в GitHub.
- Для оперативных вопросов используйте Telegram @rabb1tof.
